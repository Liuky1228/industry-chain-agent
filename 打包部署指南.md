## 产业链分析 Agent - 打包部署指南

本指南面向零基础用户，手把手教你如何将项目打包并部署到新电脑上。

---

### 一、你需要知道的事

这个项目由两部分组成：

**后端**（Python/FastAPI）：负责数据抓取、AI分析、报告生成。需要 Python 3.10+。

**前端**（React/Vite）：负责网页界面展示。需要 Node.js 18+。

打包的本质是：把源代码（约 1-2MB）拷贝过去，在新电脑上重新安装依赖（约 500MB），而不是直接拷贝整个文件夹（900多MB）。

---

### 二、打包步骤（在你的电脑上操作）

#### 第1步：运行清理脚本

双击项目根目录下的 `clean_for_dist.bat`。

它会自动删除以下内容（这些都是可以重新生成的，不影响功能）：

| 目录 | 大小 | 说明 |
|------|------|------|
| `venv\` | ~386MB | Python 虚拟环境，新电脑上会重建 |
| `frontend\node_modules\` | ~125MB | 前端依赖包，新电脑上会重建 |
| `data\` | ~406MB | 运行时数据（PDF、报告、数据库），属于本地数据 |
| `__pycache__\` | <1MB | Python 缓存文件 |

脚本还会提示你是否删除 `.env` 文件。**建议删除**，因为里面有你的 API Key（相当于密码）。

#### 第2步：压缩文件夹

清理完成后，整个项目大约 1-2MB。用以下方式之一压缩：

**方法A（推荐）：** 右键点击 `industry-chain-agent` 文件夹 → 发送到 → 压缩(zipped)文件夹

**方法B：** 如果你有 7-Zip 或 WinRAR，右键 → 添加到压缩包 → 选择 ZIP 格式

压缩后大约 500KB - 1MB。

#### 第3步：传输

把 zip 文件通过U盘、微信、邮件、网盘等方式发送到目标电脑。

---

### 三、在新电脑上部署（在目标电脑上操作）

#### 第1步：安装必要软件

目标电脑需要先安装两个软件：

**Python 3.10+**
1. 打开浏览器，访问 https://www.python.org/downloads/
2. 点击黄色的 "Download Python 3.x.x" 按钮下载安装
3. **关键：安装界面底部勾选 "Add Python to PATH"**（非常重要！不勾选的话命令行找不到 Python）
4. 点击 "Install Now" 完成安装

**Node.js 18+**
1. 打开浏览器，访问 https://nodejs.org/
2. 下载 LTS（长期支持）版本安装
3. 一路点"下一步"完成安装，默认选项即可

**验证安装是否成功：**
按 `Win + R`，输入 `cmd`，回车，在弹出的黑色窗口中输入：
```
python --version
node --version
```
如果分别显示版本号（如 `Python 3.12.0` 和 `v20.11.0`），说明安装成功。

#### 第2步：解压项目

把 zip 文件解压到你想放的位置，比如 `D:\industry-chain-agent`。

注意：路径中**不要有中文或空格**，避免后续出问题。推荐放在磁盘根目录下的英文文件夹。

#### 第3步：一键安装并启动

双击项目文件夹里的 `setup_and_start.bat`。

这个脚本会自动完成以下所有事情：
1. 检查 Python 和 Node.js 是否已安装
2. 创建 Python 虚拟环境
3. 安装所有 Python 依赖包
4. 创建 `.env` 配置文件并打开记事本让你填写 API Key
5. 安装前端依赖
6. 启动后端和前端服务

首次运行需要联网下载依赖，大约 3-10 分钟，取决于网速。

#### 第4步：配置 API Key

首次运行时，脚本会自动打开记事本让你编辑 `.env` 文件。你需要填写 SiliconFlow 的 API Key：

```
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
```

**获取 API Key 的步骤：**
1. 打开 https://cloud.siliconflow.cn
2. 注册并登录账号
3. 进入控制台 → API 密钥 → 创建新密钥
4. 复制密钥，粘贴到 `.env` 文件中 `DEEPSEEK_API_KEY=` 后面（替换掉示例文字）
5. 保存并关闭记事本

#### 第5步：开始使用

启动成功后，打开浏览器访问：

**http://localhost:3004**

这就是系统的主界面了。

---

### 四、日常使用

**启动系统：** 双击 `setup_and_start.bat`（首次会安装，之后直接启动）

**停止系统：** 关闭两个黑色命令行窗口（后端窗口和前端窗口），或者按 `Ctrl+C`

**查看 API 文档：** 浏览器打开 http://localhost:8004/docs

### 五、常见问题

**Q: 提示"python 不是内部或外部命令"**
A: Python 没有加入系统环境变量。重新安装 Python，安装时务必勾选 "Add Python to PATH"。如果不想重装，可以手动添加：右键"此电脑" → 属性 → 高级系统设置 → 环境变量 → Path → 编辑 → 新建 → 添加 Python 安装路径（如 `C:\Users\你的用户名\AppData\Local\Programs\Python\Python312\`）和 `Scripts` 子目录。

**Q: 提示"node 不是内部或外部命令"**
A: 同上，Node.js 没有加入环境变量。重新安装 Node.js 即可。

**Q: pip install 很慢或失败**
A: 网络问题。可以换用国内镜像源：
```
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**Q: npm install 很慢或失败**
A: 同样换国内镜像：
```
cd frontend
npm install --registry=https://registry.npmmirror.com
```

**Q: 端口被占用（如 8004 或 3004）**
A: 可能上次没有正常关闭。运行 `restart_backend.bat` 可以自动清理并重启后端。或者手动操作：
```
netstat -ano | findstr :8004
taskkill /PID 找到的PID /F
```

**Q: 浏览器打开 localhost:3004 显示空白或报错**
A: 检查后端是否正常运行。打开 http://localhost:8004/api/health ，如果返回 JSON 数据说明后端正常，问题在前端；如果打不开说明后端没启动成功，检查命令行窗口的报错信息。

**Q: 想保留之前的分析数据**
A: `data\` 文件夹里是运行时数据（数据库、PDF、报告）。如果你想在换电脑时保留这些数据，打包前不要运行 `clean_for_dist.bat`，而是手动删除 `venv\` 和 `frontend\node_modules\` 这两个目录即可（只删这两个就能省 500MB，保留 data 目录）。

### 六、项目文件说明

供参考，以下是项目中各文件/目录的用途：

| 文件/目录 | 用途 | 打包时是否保留 |
|-----------|------|----------------|
| `app\` | 后端 Python 源代码 | 保留 |
| `frontend\src\` | 前端 React 源代码 | 保留 |
| `frontend\index.html` | 前端入口 HTML | 保留 |
| `frontend\package.json` | 前端依赖清单 | 保留 |
| `frontend\vite.config.js` | 前端构建配置 | 保留 |
| `requirements.txt` | Python 依赖清单 | 保留 |
| `.env.example` | 配置模板（不含密钥） | 保留 |
| `start.bat` / `start.sh` | 原始启动脚本 | 保留 |
| `setup_and_start.bat` | 一键安装启动脚本（新增） | 保留 |
| `clean_for_dist.bat` | 打包清理脚本（新增） | 保留 |
| `restart_backend.bat` | 后端重启脚本 | 保留 |
| `venv\` | Python 虚拟环境 | 删除（可重建） |
| `frontend\node_modules\` | 前端依赖包 | 删除（可重建） |
| `data\` | 运行时数据 | 按需保留或删除 |
| `__pycache__\` | Python 缓存 | 删除（可重建） |
| `.env` | 含真实密钥的配置 | 建议删除 |
