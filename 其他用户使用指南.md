# 其他用户使用指南（零基础版）

> 这份文档是给**第一次用、什么都不会**的人看的。
> 目标：你跟着一步一步做，就能在 **WorkBuddy** 里让「产业链分析助手」帮你生成一份产业研究报告（Word 文档）。
> 你不需要懂编程。只要会开浏览器、装软件、复制粘贴，就行。

---

## 一、先弄懂几个词（看不懂没关系，有个印象即可）

| 词 | 大白话解释 |
|---|---|
| **WorkBuddy** | 一个 AI 助手软件。你跟它打字聊天，它能帮你干活。这份技能就装在它里面。 |
| **技能 / Skill** | 一套"操作说明书"。装了之后，WorkBuddy 就知道怎么去生成产业报告。它本身不是程序，是指令。 |
| **后端 / 引擎** | 真正干活的那个程序。技能只是说明书，后端才是跑起来算报告的"发动机"。它装在你自己电脑上，在后台运行。 |
| **终端 / 命令行** | 一个黑框框窗口，你在这里输入英文命令让电脑执行。Windows 上叫"命令提示符/PowerShell"，Mac 上叫"终端"。本文会给你现成命令，照抄回车即可。 |
| **Python** | 后端程序是用它写的，必须装。装一次就行。 |
| **Git** | 一个下载工具，用来把后端的代码从网上拉到你电脑。装一次就行。 |
| **Node.js** | 前端网站程序是用它写的（它自带 npm）。v1.2.0 起想用 `localhost:3004` 网站界面就必须装；不装也能出报告，只是没有网站。装一次就行。 |
| **前端 / 网站** | 一个图形界面（网址 `localhost:3004`）。你在上面点按钮就能发起任务、看进度、下报告，比在聊天里拿链接更直观。它由 Node.js 跑起来。 |
| **API Key（密钥）** | 一串像密码的字符串（以 `sk-` 开头）。程序靠它去调用 AI 来写报告。**必须填你自己的**，相当于你自己的"饭票"，别人不能替你付。 |
| **GitHub** | 一个存放代码的地方。后端代码存在发布者的 GitHub 仓库里，你的电脑会从那里下载。 |

**你真正要做的三件事：**
1. 在 WorkBuddy 里装上这个技能（点几下鼠标）。
2. 在终端里跑一条命令，把"发动机"下载安装好，并填上你自己的密钥。
3. 在 WorkBuddy 里打一句话，等几分钟，拿到报告。

下面开始。

---

## 第 0 步：先确认发布者已经准备好（你只需看一眼）

这份技能能正常用，**前提是发布者（把这个技能做出来的人）已经做好了下面两件事**：

- ✅ 已经把这个技能发布到了 SkillHub（你能搜到、能装）。
- ✅ 已经把后端的 GitHub 仓库设成了 **Public（公开）**。

> 如果你在第 5 步卡在"下载/克隆失败、找不到仓库"，先联系发布者确认这两点。你作为使用者不用去设这些。

另外，**你需要自己准备一个 AI 密钥（API Key）**——下面第 6 步会手把手教你怎么拿。

---

## 第 1 步：安装 WorkBuddy（如果你已经有了，跳过）

1. 打开浏览器，搜"WorkBuddy"或问发布者要下载地址。
2. 下载安装包，双击安装，一路"下一步"。
3. 安装完打开，能正常进入聊天界面就行。

---

## 第 2 步：在 SkillHub 上安装「产业链分析助手」技能

1. 打开 WorkBuddy。
2. 在左侧边栏找到 **「技能中心」/「技能市场」/「Skills」**（不同版本名字可能略有不同，找带"技能"二字的入口）。
3. 在搜索框里输入：`industry-chain-agent` 或中文 `产业链分析助手`。
4. 找到后点 **「安装 / Install」**。
5. 等几秒，出现"已安装"就成功了。

> 装完之后，技能文件会出现在你电脑上一个隐藏文件夹里（`C:\Users\你的用户名\.workbuddy\skills\...`）。**你不用去翻这个文件夹**，后面第 5 步会直接给你命令自动进去。

---

## 第 3 步：安装三个必要软件（只需装一次）

后端程序需要 **Python**、**Git**，如果想用网站界面（`localhost:3004`）还需要 **Node.js**。按你的系统装：

### 🪟 如果你用 Windows

**A. 装 Python**
1. 浏览器打开 `https://www.python.org/downloads/`。
2. 点那个大大的 **Download Python 3.x.x** 按钮（下最新版即可，3.10 以上都行，推荐 3.11 或 3.12）。
3. 双击下载好的安装包。
4. ⚠️ **最关键一步**：安装界面最下面，一定一定要勾选 **「Add Python to PATH」**（把 Python 添加到环境变量），然后再点「Install Now」。
5. 等它装完，点 Close。

**B. 装 Git**
1. 浏览器打开 `https://git-scm.com/download/win`。
2. 下载 64-bit 版本，双击安装。
3. 一路点 "Next / 下一步"，**全部用默认选项**即可，最后点 Install。

### 🍎 如果你用 Mac

**A. 装 Python**
1. 打开"终端"（见第 4 步知道怎么开）。
2. 先试一下有没有：输入 `python3 --version` 回车。
   - 如果显示 `Python 3.x.x` → 已经有了，跳过。
   - 如果提示找不到 → 去 `https://www.python.org/downloads/macos/` 下载 Mac 版安装，双击 pkg 一路下一步。
   - （更省事的办法：装了 Homebrew 的话直接 `brew install python@3.12`，但新手建议用官网安装包。）

**B. 装 Git**
1. 打开"终端"，输入 `git --version` 回车。
   - 如果显示 `git version x.x.x` → 已有，跳过。
   - 如果弹窗问要不要安装"命令行开发者工具"，点 **「安装」**，等它装完。
   - 否则去 `https://git-scm.com/download/mac` 下载安装。

**C. 装 Node.js（Windows 和 Mac 都要，想用网站才需要）**
1. 浏览器打开 `https://nodejs.org/`，下载 **LTS（长期支持版）** 那个大按钮对应的安装包。
2. Windows：双击安装包一路"下一步"；Mac：双击 `.pkg` 一路"继续"。装完会自动带上 `npm`。
3. 验证（Windows 在 cmd、Mac 在终端输入）：
   ```
   node --version
   npm --version
   ```
   能看到 `v18.x.x` 以上 和 `npm x.x.x` 就成功了。
> 只想在 WorkBuddy 聊天里拿报告、不要网站界面？Node.js 可以不装，跳过这步也行。

### ✅ 验证装好了没（重要）

**Windows**：按 `Win` 键，输入 `cmd` 回车，打开黑窗口，分别输入下面两行（每行输完按回车）：
```
python --version
git --version
node --version
```
能看到类似 `Python 3.12.1`、`git version 2.xx.x`、`v18.x.x` 就成功了（没装 Node 就忽略最后一行）。

> 如果 `python --version` 提示"不是内部或外部命令"，说明第 3 步 A 的第 4 步没勾选 PATH——重新运行 Python 安装包，选 "Modify"，把 "Add Python to PATH" 勾上再装一次。

**Mac**：打开"终端"，输入：
```
python3 --version
git --version
node --version
```
能看到版本号就 OK（没装 Node 就忽略最后一行）。

## 第 4 步：打开"终端 / 命令行"

后面要在这里敲命令。

- **Windows**：按 `Win` 键，输入 `cmd`（或"命令提示符"），按回车。出来一个黑底（或白底）窗口，光标在闪，就是它。
- **Mac**：按 `Command + 空格`，输入 `终端` 或 `terminal`，回车。出来一个窗口就是它。

> 小贴士：在终端里，**复制命令后右键=粘贴**（Mac 用 `Command+V`）。输完一行按 **回车** 执行。不要自己手打那些长路径，容易错，复制粘贴最稳。

---

## 第 5 步：下载并安装后端（最关键的一步）

现在让电脑把"发动机"下载下来并装好。

**Windows（在刚才的 cmd 黑窗口里，一行一行来）：**
```
cd %USERPROFILE%\.workbuddy\skills\industry-chain-agent
python scripts\install.py
```
> 说明：`%USERPROFILE%` 会自动变成你电脑的用户目录（比如 `C:\Users\张三`），不用你手打用户名。

**Mac / Linux（在终端里）：**
```
cd ~/.workbuddy/skills/industry-chain-agent
python3 scripts/install.py
```

**敲下回车后会发生什么（正常情况）：**
1. 屏幕开始刷字，先 `克隆后端仓库 ...`，就是从网上把代码下载到你电脑。
2. 然后 `创建虚拟环境 ...`、`安装依赖 ...`——这一步要下载一堆 Python 零件，**可能要等 1～5 分钟**，期间窗口在动、在下载，是正常现象。（如果你装了 Node.js，它还会顺带 `安装前端依赖 (npm install)`，网站界面就是这时准备好的，多等一两分钟。）
3. 最后会看到类似：
   ```
   [安装] 已生成 C:\Users\你的用户名\.industry-chain-agent\.env
   ⚠️ 请编辑该文件，填入你自己的 DEEPSEEK_API_KEY，然后重新运行本技能。
   ```
   看到这行就说明**下载安装成功了**，它已经帮你建好了一个 `.env` 文件，接下来只差填密钥。

> ⚠️ **不要关掉这个黑窗口先**，第 6 步还要用类似的方式（或者直接用记事本）去填密钥。
> 如果卡在 `克隆后端仓库` 很久报错（比如 `Could not resolve host` / `Connection reset` / `Permission denied`），说明你的网络访问 GitHub 不畅，看文末"常见问题"里的代理/SSH 办法，或者直接联系发布者。

## 第 6 步：拿到并填写你自己的密钥（最不能漏的一步）

报告是用 AI 写的，AI 要"饭票"才能干活，这个饭票就是 **API Key**。

### 6.1 去哪里拿密钥

1. 打开浏览器，进入**阿里云百炼控制台**：`https://bailian.console.aliyun.com/`（或 DashScope：`https://dashscope.aliyun.com/`）。
2. 用你的**阿里云账号**登录（没有就注册一个，需要实名认证）。
3. 在页面里找到 **「API Key 管理」**（百炼里左侧菜单；DashScope 里右上角头像 → API-KEY）。
4. 点 **「创建 API Key」** / 「创建密钥」。
5. 生成后会出现一串以 **`sk-`** 开头的字符串，**立刻复制保存**（密钥通常只显示这一次）。
   - 形如：`sk-9a8b7c6d5e4f3g2h1i0j...`（后面很长）。
   - 这就是你的"饭票"，**不要发给别人**。

> 费用提示：阿里云百炼新账号通常有免费额度，用超了按量计费。生成一份产业报告会消耗一些额度，但一般测试几份是够的。

### 6.2 把密钥填进 `.env` 文件

刚才第 5 步已经生成了一个 `.env` 文件，我们要找出来、改一行。

**🪟 Windows 做法（用记事本，最稳）：**

1. 按 `Win + E` 打开"文件资源管理器"。
2. 在顶部地址栏**整行清空**，输入 `%USERPROFILE%`，按回车 → 进入你的用户目录。
3. 点顶部「查看」菜单，勾选 **「隐藏的项目」**（这样才能看到带点的文件夹）。
4. 双击进入 `.industry-chain-agent` 文件夹。
5. 找到文件 **`.env`**（可能显示类型为"ENV 文件"）。
6. 在它上面**右键 → 打开方式 → 记事本**。
7. 记事本打开后，你会看到几行字，找到这一行：
   ```
   DEEPSEEK_API_KEY=sk-请在此填写你自己的密钥
   ```
8. 把 `=` 后面的 `sk-请在此填写你自己的密钥` **整个删掉**，换成**你刚复制的真实密钥**（真实密钥本身就以 `sk-` 开头，直接整段粘贴上去即可）。
   - 改完后这一行应该是：`DEEPSEEK_API_KEY=sk-你真实的密钥一大串`
   - 其它行（BASE_URL、MODEL 等）**一个字都不要动**。
9. 按 `Ctrl + S` 保存，关闭记事本。

> ⚠️ 坑点：Windows 记事本如果让你"另存为"，**保存类型一定要选"所有文件"，文件名保持 `.env`**，千万别存成 `.env.txt`！因为我们是"打开已有的 .env 再保存"，一般不会有这问题，放心。

**🍎 Mac 做法（用文本编辑）：**

1. 打开"访达（Finder）"，按 `Shift + Command + G`，弹出"前往文件夹"。
2. 输入 `~/.industry-chain-agent`，点"前往"（这样能看到隐藏文件夹）。
3. 找到 `.env` 文件，**右键 → 打开方式 → 文本编辑**。
4. 文本编辑里，先点顶部「格式 → 制作纯文本」（避免变成富文本）。
5. 找到 `DEEPSEEK_API_KEY=sk-请在此填写你自己的密钥`，把 `=` 后面整段换成你的真实密钥。
6. 按 `Command + S` 保存。如果提示"纯文本"转换，点"好/转换"。
   > ⚠️ 坑点：保存后确认文件名是 `.env` 不是 `.env.txt`。若不小心成了 `.env.txt`，在访达里右键"重新命名"改成 `.env`。

填好保存后，这一步就完成了。

## 第 7 步：在 WorkBuddy 里生成你的第一份报告

1. 打开 WorkBuddy，**新开一个对话**（点"新建对话 / +"）。
2. 在输入框里打字（照抄下面任一句，换成你想分析的行业）：
   ```
   我想分析光伏产业链
   ```
   或者：`做一份新能源汽车产业报告` / `跑一下半导体产业链`
3. 按回车发送。
4. 接下来 WorkBuddy 会自己忙活：
   - 先"健康检查"，确认后端就位；
   - 然后**在后台启动发动机（后端）**，你可能看到闪过一个黑窗口（Windows）或终端在跑，不用管它；
   - 接着它去网上搜集研报、用 AI 抽取企业关系、生成报告。这个过程**通常要几分钟到十几分钟**，请耐心等，别关窗口。
5. 完成后，它会在聊天里给你回复，包含：
   - 一句摘要（产业名、涉及多少家企业、多少条传导关系）；
   - 一个**下载链接**。

> 如果它回复说"未安装 / 请先运行 install.py / 缺密钥"——回到第 5、6 步检查是不是漏了哪一步。

## 第 8 步：拿到报告后怎么看

1. 在 WorkBuddy 的回复里，找到那个链接，类似：
   ```
   http://localhost:8004/api/tasks/任务ID/report
   ```
2. **直接点这个链接** → 浏览器会下载一个文件，文件名类似 `产业链分析报告_光伏.docx`。
3. 下载位置一般在你电脑的 **「下载 / Downloads」** 文件夹。
4. 用 **Word** 或 **WPS** 打开这个 `.docx` 文件，就是一份完整的、8 章的机构风格产业研究报告。

> 报告文件同时也在你电脑的 `~/.industry-chain-agent/data/` 目录下存着一份，想直接翻文件夹找也行。

✅ **v1.2.0 起新增网站界面**：除了下载 Word，你还能在浏览器打开 `http://localhost:3004` 使用完整网站（发起任务 / 看进度 / 历史记录 / 下载报告）。前提是本机装了 **Node.js ≥ 18**（第 3 步），且前后端都启动了。下面第 9 步教你怎么开。

## （可选）第 9 步：用网站界面（localhost:3004）

如果你更习惯"网页点一点"而不是在聊天里拿链接，可以用这个免费附赠的网站：

1. 在终端里进入技能目录（和上面第 5 步一样）：
   - Windows：`cd %USERPROFILE%\.workbuddy\skills\industry-chain-agent`
   - Mac：`cd ~/.workbuddy/skills/industry-chain-agent`
2. 运行（会同时启动前端 3004 和后端 8004 两个服务）：
   - Windows：`python scripts\start_site.py`
   - Mac：`python3 scripts/start_site.py`
3. 等大约 12 秒，浏览器打开 `http://localhost:3004`，就能看到网站了。
4. 在网站里输入产业名 → 发起任务 → 实时看进度 → 在历史里下载 Word 报告。
5. 用完想关掉：在终端按 `Ctrl + C`，或另开一个终端运行 `python scripts\stop_site.py`（Mac 用 `python3`）。

> 注意：网站和后端要**同时运行**（前端占 3004、后端占 8004）。只想在 WorkBuddy 聊天里拿报告、不用网站，跳过这步也完全没问题。

## 常见问题 / 排错

**Q1：第 5 步卡在"克隆后端仓库"，报错 Connection reset / Could not resolve host / Permission denied**
→ 你的网络访问 GitHub 不畅（国内常见）。两个办法，二选一：
- **办法 A（用代理，最简单）**：如果你平时用代理/VPN 上网，先在终端设置代理再跑安装（把 `7890` 换成你代理软件的端口）：
  ```
  git config --global http.proxy http://127.0.0.1:7890
  ```
  然后重新跑第 5 步的 `python scripts\install.py`。
- **办法 B（SSH，稍复杂）**：需要你自己在 GitHub 官网注册账号、生成 SSH 密钥并加到账号里，然后在终端执行：
  ```
  git config --global "url.ssh://git@github.com/.insteadOf" "https://github.com/"
  ```
  再重新跑安装命令。
- 都不行就联系发布者，让他把后端打包发你。

**Q2：输入 `python` 提示"不是内部或外部命令"**
→ Python 没装好或没勾 PATH。重装 Python，记得勾 **Add Python to PATH**（第 3 步）。Windows 也可以试把 `python` 换成 `py`：`py scripts\install.py`。

**Q3：WorkBuddy 说"缺密钥 / health_check 返回 5"**
→ `.env` 里的 `DEEPSEEK_API_KEY` 没填或填错。回第 6 步，确认 `.env` 里那一行是 `DEEPSEEK_API_KEY=sk-你的真实密钥`，且没多余空格、没存成 `.env.txt`。

**Q4：启动时报"端口 8004 被占用"**
→ 你电脑上别的地方已经占用了 8004（比如你也跑了发布者原来的项目后端）。关掉那个程序/窗口，再重新在 WorkBuddy 里发一句触发。

**Q5：报告一直卡在 90%（整合阶段）**
→ 这是已知小问题。去看后端日志（黑窗口里的报错），或直接重新发一句让它重试。

**Q6：Mac 上 `python` 命令不存在**
→ Mac 用 `python3`。第 5、7 步里所有 `python` 都换成 `python3`。

**Q7：想用网站（localhost:3004）但打不开**
→ 两种可能：① 没装 Node.js（第 3 步 C），装好重跑 `install.py` 让它执行 `npm install`；② 没运行 `start_site.py`，按第 9 步在终端跑它，等约 12 秒再开网页。只想要报告的话，用聊天里的下载链接即可，网站不是必须的。
